## Report Types

### 1. Production Validation Report
**File:** `production-validation-YYYY-MM-DD.md`  
**Purpose:** Comprehensive validation of all production readiness gates

**Includes:**
- Build verification
- Test execution results
- HYPER_STANDARDS compliance
- Database configuration
- Environment variables
- Security hardening
- Performance baselines
- Multi-cloud readiness
- Health checks
- Rollback plan

### 2. Security Audit Summary
**File:** `security-audit-YYYY-MM-DD.md`  
**Purpose:** Security-focused code and dependency analysis

**Includes:**
- Hardcoded credential scan
- Mock/stub detection
- Library CVE assessment
- Security recommendations
- Compliance matrix
- Penetration test recommendations

### 3. Validation Summary
**File:** `VALIDATION-SUMMARY.md` (always latest)  
**Purpose:** Executive summary for quick status check

**Includes:**
- Overall pass/fail status
- Critical blockers
- What was validated
- What could not be validated
- Recommended path forward
- Risk assessment

## Validation Gates

Production deployment requires ALL gates to PASS:

1. ✅ Build successful (no errors)
2. ✅ All tests passing (0 failures)
3. ✅ HYPER_STANDARDS clean (0 violations)
4. ✅ Database configured and migrated
5. ✅ Environment variables set (no hardcoded values)
6. ✅ WAR files build successfully
7. ✅ Security hardening complete
8. ✅ Performance baselines met
9. ✅ Docker/K8s configs valid
10. ✅ Health checks operational

## How to Run Validation

### Automated (Recommended)
```bash
# From repository root
claude "Validate production readiness"
# OR
cd .claude && ./skills/yawl-validate.sh --production
```

### Manual
```bash
# 1. Build verification
mvn clean package -Pprod

# 2. Test verification
mvn clean test
# Check: 0 failures, 0 errors

# 3. HYPER_STANDARDS check
grep -rn "TODO\|FIXME\|XXX\|HACK" src/
grep -rn "mock\|stub\|fake" src/ --include="*.java"
# Expected: 0 results

# 4. Security scan
mvn org.owasp:dependency-check-maven:check -Pprod

# 5. Performance baseline
./scripts/run-performance-tests.sh

# 6. Health checks
docker-compose up -d
curl http://localhost:8080/health
```

## Rollback Criteria

Automatic rollback triggered by:
- Any test failures
- HYPER_STANDARDS violations
- Security vulnerabilities (CVSS > 7.0)
- Performance degradation > 20%
- Health checks failing
