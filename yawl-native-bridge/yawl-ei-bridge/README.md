# YAWL Erlang Interface Bridge (EI Bridge)

Implementation of JVM↔BEAM ei bridge for Erlang distribution protocol with Unix domain socket transport.

## Architecture

### Layer 1: jextract Bindings
- **Location**: `headers/ei.h`
- **Purpose**: Generate C bindings from Erlang's erl_interface library
- **Generated by**: `jextract` during Maven build
- **Package**: `org.yawlfoundation.yawl.nativebridge.erlang.generated`

### Layer 2: ErlangNode
- **Classes**: `ErlangNode`, `ErlTerm` and subclasses
- **Purpose**: Type-safe bridge between Java and Erlang
- **Key Features**:
  - Connection management with Unix domain sockets
  - RPC operations with timeout support
  - Type-safe Erlang term encoding/decoding
  - Thread-safe connection pooling

### Layer 3: ProcessMiningClient
- **Interfaces**: `ProcessMiningClient`, `ProcessMiningClientImpl`
- **Purpose**: Domain API for process mining operations
- **Operations**: Process discovery, conformance checking, performance analysis

## Key Components

### ErlTerm Hierarchy
```java
sealed interface ErlTerm
    permits ErlAtom, ErlList, ErlTuple, ErlBinary, ErlLong

// Examples
ErlAtom atom("hello")          // Erlang atom
ErlLong longValue(42)          // Erlang integer
ErlBinary.fromString("test")   // Erlang binary
ErlList.of(items)              // Erlang list
ErlTuple.of(items)             // Erlang tuple
```

### ErlangNode
```java
// Create connection
ErlangNode node = new ErlangNode("yawl@localhost", "secret", (short) 1);
node.connect();

// Execute RPC
ErlTerm result = node.rpc("module", "function", arguments);

// Close connection
node.close();
```

### ProcessMiningClient
```java
// Create client
ProcessMiningClient client = new ProcessMiningClientImpl(node);

// Discover process model
List<EventLogEntry> eventLog = ...;
ErlTerm model = client.discoverProcessModel(eventLog);

// Conformance checking
ConformanceResult result = client.conformanceCheck(model, eventLog);

// Performance analysis
PerformanceResult perf = client.analyzePerformance(model, eventLog);
```

## Usage Examples

### Basic Connection
```java
public class BasicUsage {
    public static void main(String[] args) {
        try {
            // Create Erlang node connection
            ErlangNode node = new ErlangNode("yawl@localhost", "secret");
            node.connect();

            // Create simple terms
            ErlTerm atom = ErlAtom.atom("hello");
            ErlTerm number = ErlLong.longValue(42);

            // Execute RPC call
            ErlTerm result = node.rpc("math", "add",
                ErlList.of(number, number));

            System.out.println("Result: " + result);

        } catch (ErlangException e) {
            System.err.println("Erlang error: " + e.getMessage());
        }
    }
}
```

### Process Mining Example
```java
public class ProcessMiningExample {
    public static void main(String[] args) {
        try {
            // Create event log
            List<EventLogEntry> eventLog = Arrays.asList(
                new EventLogEntry("case1", "start", 1234567890L,
                    Map.of("user", "john")),
                new EventLogEntry("case1", "task1", 1234567950L,
                    Map.of("user", "john", "department", "sales")),
                new EventLogEntry("case1", "end", 1234568000L,
                    Map.of("user", "john"))
            );

            // Create client
            ErlangNode node = new ErlangNode("pm@localhost", "yawl-pm");
            ProcessMiningClient client = new ProcessMiningClientImpl(node);

            // Discover process model
            ErlTerm model = client.discoverProcessModel(eventLog);
            System.out.println("Discovered model: " + model);

            // Validate model
            ValidationResult validation = client.validateProcessModel(model);
            if (validation.isValid()) {
                System.out.println("Model is valid");
            } else {
                System.out.println("Validation errors: " + validation.getErrors());
            }

        } catch (ErlangException e) {
            System.err.println("Process mining error: " + e.getMessage());
        }
    }
}
```

## Performance Considerations

### Connection Management
- Reuse ErlangNode instances when possible
- Close connections when no longer needed
- Use connection pooling for high-throughput scenarios

### Memory Management
- ErlTerms are immutable and thread-safe
- Large binaries should be handled carefully
- Use streaming for large event logs

### Thread Safety
- ErlangNode is thread-safe for concurrent RPC calls
- ErlTerms are immutable and can be shared across threads
- ProcessMiningClient delegates to ErlangNode, making it thread-safe

## Troubleshooting

### Common Issues
1. **Connection Failed**
   - Check Erlang node is running
   - Verify node name and cookie match
   - Ensure erl_interface is installed

2. **Encoding Errors**
   - Check Erlang term types match expected format
   - Verify binary encoding works correctly
   - Use toErlString() for debugging

3. **RPC Timeouts**
   - Increase timeout values for complex operations
   - Check network connectivity
   - Monitor Erlang node performance
