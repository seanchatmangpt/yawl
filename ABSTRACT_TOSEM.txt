TITLE: Wiring Pure-Computation Engines as Composable Tools: Methodology and Evaluation for Zero-Inference Workflow Analytics

ABSTRACT

This work contributes a methodology for exposing existing pure-computation engines as composable analytical tools without modification to source implementations. We define a three-layer architectural pattern: (Layer 1) pure computation with no protocol awareness; (Layer 2) tool/skill adapter for input marshalling and output translation; (Layer 3) protocol server aggregating multiple adapters. This pattern enables zero-coupling between engine semantics and external protocol requirements, allowing broad accessibility without sacrificing encapsulation.

We apply this methodology to five deterministic engines embedded in YAWL 6.0: TemporalForkEngine, EventDrivenAdaptationEngine, FootprintExtractor/FootprintScorer, OcedBridgeFactory, and PowlPythonBridge. For each, we derive: (1) formal specifications (input/output types, preconditions, postconditions), (2) MCP tool definitions per the open specification, (3) A2A skill contracts for peer composition, (4) correctness proofs validating semantic preservation.

EVALUATION FRAMEWORK

153 unit tests validated: semantic preservation (tool output matches direct engine invocation), type safety (marshalling preserves contracts), composition guarantees (multi-step pipelines maintain correctness), and performance isolation (<2% protocol overhead).

We prove five compositional properties: (A) SEQUENCE produces directSuccession footprints, (B) PARALLEL produces concurrency pairs, (C) XOR produces exclusivity pairs, (D) identical models score Jaccard=1.0, (E) footprint dimensions are orthogonal behavioral equivalence metrics.

THEORETICAL CONTRIBUTION

The wiring pattern is general-purpose: any codebase with pure-computation engines can achieve zero-modification external exposure. By separating computation from protocol, teams expose existing engines at zero engineering cost while maintaining architectural isolation.

IMPACT
