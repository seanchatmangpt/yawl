HOW TO USE THIS BASELINE
================================================================================

SCENARIO 1: Update One Library

1. Read: LIBRARY_UPDATE_PROCEDURE.md (Phase 1-2)
2. Update: One dependency in pom.xml
3. Test: mvn clean compile && mvn clean test
4. Compare: Results against baseline (174 tests should pass)
5. Verify: Coverage >= 80%, no new errors
6. Commit: With clear message referencing baseline

SCENARIO 2: Performance Regression Check

1. Establish baseline: mvn clean test (capture times)
2. Update library
3. Rerun tests: mvn clean test
4. Compare: Build time should be <20% slower
5. Load test: k6 run validation/performance/load-test.js
6. Decision: Accept if <10%, investigate if >10%, rollback if >20%

SCENARIO 3: Security Vulnerability Alert

1. Vulnerability discovered
2. Update affected library version
3. Run: mvn -Psecurity-audit clean install
4. Verify: No new CVEs introduced
5. Run: mvn clean test (should pass)
6. Deploy with confidence

SCENARIO 4: Multiple Library Updates

1. Create tracking spreadsheet
2. Update ONE library (see Scenario 1)
3. Test and commit
4. Move to next library
5. Document cumulative impact
6. Create pull request with all changes
