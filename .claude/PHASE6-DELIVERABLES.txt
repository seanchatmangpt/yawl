## H-GUARDS VALIDATION

7 Forbidden Patterns Defined:
  • H_TODO: TODO/FIXME markers
  • H_MOCK: Mock class/method names
  • H_STUB: Empty returns from non-void methods
  • H_EMPTY: No-op method bodies
  • H_FALLBACK: Silent catch-and-fake patterns
  • H_LIE: Code ≠ Javadoc documentation
  • H_SILENT: Log instead of throw exception

RDF Classes for Violations:
  • code:GuardViolation (violation instance)
  • code:Method (containing method)
  • code:SourceFile (generated file)
  • code:HyperStandardsReceipt (validation report)

## ARCHITECTURE DECISIONS

5 Key Decisions Made:
  1. Apache Jena over RDF4J for v6.0 (scale <1M triples)
  2. Async event batching with 500ms timeout (no case latency)
  3. Column-level lineage tracking (enables compliance)
  4. H-Guards as RDF citizens (query code + lineage together)
  5. SPARQL over custom query API (W3C standard)
